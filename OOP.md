```cpp
    };
```

### Constructor
- It is a special member function which is automatically called when an object of the class is created. It is used to initialize the data members of the class and it has no return type. There are many types of constructors used

       1.Default Constructor
       2.Parameterized Constructor
       3.Copy Constructor
       4.Move Constructor
       5.explicit Constructor
       6.constexpr Constructor
       7.Delegating Constructor
### 1. Default Constructor
- It is a constructor that takes no arguments and is automatically generated by the compiler if no other constructors are defined. It initializes the data members of the class to default values.
```cpp
class Person {
public:
    string name;
    int age;
    // Default Constructor
    Person() {
        name = "Unknown";
        age = 0;
    }
};
```
### 2. Parameterized Constructor
- It is a constructor that takes arguments and is used to initialize the data members of the class with specific values.
```cpp
class Person {
public:
    string name;
    int age;
    // Parameterized Constructor
    Person(string n, int a) {
        name = n;
        age = a;
    }
};
```
### 3. Copy Constructor
- It is a constructor that takes a reference to an object of the same class and is used
to create a new object as a copy of an existing object.
```cpp
class Person {
public:
    string name;
    int age;
    // Copy Constructor
    Person(const Person& other) {
        name = other.name;
        age = other.age;
    }
};
int main(){
    Person p1("Aftab", 30);
    Person p2 = p1; // Copy constructor is called
}
```
### 4. Move Constructor
- It is a constructor that takes an rvalue reference to an object of the same class and is used to transfer ownership of resources from a temporary object to a new object.
```cpp
class Person {
public:
    string name;
    int age;
    // Move Constructor
    Person(Person&& other) {
        name = std::move(other.name);
        age = other.age;
        other.age = 0; // optional: reset source object
    }
};
int main(){
    Person p1("Aftab", 30);
    Person p2 = std::move(p1); // Move constructor is called
}
```
### 5. explicit Constructor
- It is a constructor that is declared with the `explicit` keyword and is used to prevent implicit conversions from other types to the class type.
```cpp
class Person {
public:
    string name;
    int age;
    // Explicit Constructor
    explicit Person(string n, int a) {
        name = n;
        age = a;
    }
};
int main(){
    Person p1("Aftab", 30); // OK, direct initialization
    // Person p2 = "Aftab"; // Error, implicit conversion not allowed
}
```
### 6. constexpr Constructor
- It is a constructor that is declared with the `constexpr` keyword and is used to create objects that can be evaluated at compile time.
```cpp
class Person {
public:
    string name;
    int age;
    // Constexpr Constructor
    constexpr Person(string n, int a) : name(n), age(a) {}
};
int main(){
    constexpr Person p("Aftab", 30); // evaluated at compile time
}
```
### 7. Delegating Constructor
- It is a constructor that calls another constructor of the same class to initialize the data members.
```cpp
class Person {
public:
    string name;
    int age;
    // Delegating Constructor
    Person() : Person("Unknown", 0) {} // delegates to parameterized constructor
    Person(string n, int a) : name(n), age(a) {}
};
```
### Destructor
- It is a special member function which is automatically called when an object of the class goes out of scope or is explicitly deleted. It is used to release any resources that the object may have acquired during its lifetime and it also has no return type.
```cpp
class Person {
public:
    string name;
    int age;
    // Destructor
    ~Person() {
        cout << "Destructor called for " << name << endl;
    }
};
```
 `If the class manages resources (dynamic memory, files, etc.), then it must provided a destructor, and likely also a copy constructor and assignment operator (Rule of Three/Five).
But for simple cases the compiler‑generated ones are enough.`
## An early view of Member functions 
- Member functions are functions that are defined inside a class and can access the data members of the class. They are used to perform operations on the data members and to provide functionality to the objects of the class.\ 

       1.Ordinary member fuctions\
       2.Static member functions\
       3.virtual member functions\
       4.Pure virtual member functions\
       5.volatile Member Functions\
       6.constexpr Member Functions\
       7.Inline Member Functions\
       8.explicit Member Functions (Constructors & Conversion Operators)
### 1.Ordinary member functions (non-static)
- These are the most common type of member functions that operate on the data members of the class and can be called using an object of the class,also sometimes have an implicit `this` pointer.It can be `Volatile` or   `Const`.
```cpp
class Calculator{
public:
    int add(int a,int b){      // ordinary member function
        return a + b;
    }
    void print()const;          // can be const-qualified (see later)
};
int main(){
    Calculator calc;
    calc.add(3, 4);              // object.function()
    Calculator* p = &calc;
    p->add(5, 6);                // pointer->function()
}
```
### 2.Static member functions
- These are member functions that are declared with the `static` keyword and can be called without an object of the class. They can only access static data members or other static functions directly from the class.\
Cannot be `const` (no `this`), `virtual`, or `volatile`
Use for factory methods, utility functions, or managing shared state
```cpp
class IDGenerator{
    static int nextID;           // static data member
public:
    static int getNextID(){     // static member function
        return nextID++;
    }
    static void reset(int start){
        nextID=start;
    }
};
int IDGenerator::nextID= 100;   // definition outside class
int main(){
    int id1= IDGenerator::getNextID();  // call without object
    IDGenerator gen;
    int id2 =gen.getNextID();           // also possible via instance
}
```
### 3.Virtual member functions
- These are member functions that are declared with the `virtual` keyword and can be overridden in derived classes. They are used to achieve polymorphism in C++.\
keyPoints:  `virtual` in base, `override` in derived  classes,dynamic dispatch via pointer/reference,cannot be static and destructors should be virtual if class is meant to be inherited
```cpp
class Shape {
public:
    virtual double area()const = 0;   // pure virtual
    virtual void draw()const = 0;
    virtual ~Shape()=default;
};

class Circle:public Shape{
    double r;
public:
    Circle(double radius):r(radius){}
    double area()const override{return 3.14 * r * r;}
    void draw()const override{cout<<"Drawing Circle\n"; }
};

// Shape s; // Error ,abstract class
Circle c(5); // this is correct
```
- Class becomes abstract (cannot be instantiated),`pure virtual functions` can have a body (but still =0), but rarely done.
### 4.Pure virtual member functions
- These are member functions that are declared with the `virtual` keyword and assigned a value of `0`. They do not have an implementation in the base class and must be overridden in derived classes. They are used to create abstract classes in C++.\
```cpp
class Shape {
public:
    virtual double area()const = 0;   // pure virtual
    virtual void draw()const = 0;
    virtual ~Shape()=default;
};
```
### 5.volatile Member Functions
- These are member functions that are declared with the `volatile` keyword and can be called on volatile objects. They indicate that the function may modify the object in ways that are not predictable by the compiler, such as through hardware or concurrent access.
```cpp
class Sensor {
public:
    volatile int value; // volatile data member
    void read() volatile { // volatile member function
        // read sensor value, may change unexpectedly
    }
};
```
### 6.constexpr Member Functions
- These are member functions that are declared with the `constexpr` keyword and can be evaluated at compile time. They must have a body that consists of a single return statement and cannot modify any data members.
```cpp
class Point {
public:
    int x, y;
    constexpr Point(int xVal, int yVal) : x(xVal), y(yVal) {} // constexpr constructor
    constexpr double distanceFromOrigin() const { // constexpr member function
        return sqrt(x * x + y * y);
    }
};
```
### 7.Inline Member Functions
- These are member functions that are defined inside the class definition. The compiler may choose to expand the function inline at the point of call, which can improve performance for small functions.
```cpp
class Math {
public:
    int add(int a, int b) { return a + b; } // inline member function
};
```
### 8.explicit Member Functions (Constructors & Conversion Operators)
- These are member functions that are declared with the `explicit` keyword and are used to prevent implicit conversions from other types to the class type. This is typically used for constructors and conversion operators.
```cpp
class Person {
public:
    string name;
    int age;
    explicit Person(string n, int a) : name(n), age(a) {} // explicit constructor
    explicit operator string() const { return name; } // explicit conversion operator
};
int main(){
    Person p1("Aftab", 30); // OK, direct initialization
    // Person p2 = "Aftab"; // Error, implicit conversion not allowed
    string name = static_cast<string>(p1); // OK, explicit conversion
}
```

## 2.Inheritance
- It is the process by which one class (called derived class) can inherit properties and behaviours from another class (called base class). It allows for code reusability and the creation of a hierarchical relationship between classes.
```cpp
class Animal {
public:
    void eat() {
        cout << "Eating...\n";
    }
};
class Dog : public Animal { // Dog inherits from Animal
public:
    void bark() {
        cout << "Barking...\n";
    }
};
int main() {
    Dog myDog;
    myDog.eat(); // Inherited from Animal
    myDog.bark(); // Defined in Dog
}
```
### Types of Inheritance 
       1.Single Inheritance
       2.Multiple Inheritance
       3.Multilevel Inheritance
       4.Hierarchical Inheritance
       5.Hybrid Inheritance
### 1.Single Inheritance
- In this type of inheritance, a derived class inherits from a single base class.
```cpp
class Vehicle {
public:
    void start() {
        cout << "Vehicle starting...\n";
    }
};
class Car : public Vehicle { // Car inherits from Vehicle
public:
    void drive() {
        cout << "Car driving...\n";
    }
};
int main() {
    Car myCar;
    myCar.start(); // Inherited from Vehicle
    myCar.drive(); // Defined in Car
}
```
### 2.Multiple Inheritance
- In this type of inheritance, a derived class inherits from more than one base class.
```cpp
class Engine {
public:
    void startEngine() {
        cout << "Engine starting...\n";
    }
};
class Wheels {
public:
    void rotateWheels() {
        cout << "Wheels rotating...\n";
    }
};
class Car : public Engine, public Wheels { // Car inherits from both Engine and Wheels
public:
    void drive() {
        cout << "Car driving...\n";
    }
};
int main() {
    Car myCar;
    myCar.startEngine(); // Inherited from Engine
    myCar.rotateWheels(); // Inherited from Wheels
    myCar.drive(); // Defined in Car
}
```
### 3.Multilevel Inheritance
- In this type of inheritance, a derived class inherits from a base class, and then another class inherits from that derived class.
```cpp
class Animal {
public:
    void eat() {
        cout << "Eating...\n";
    }
};
class Mammal : public Animal { // Mammal inherits from Animal
public:
    void breathe() {
        cout << "Breathing...\n";
    }
};
class Dog : public Mammal { // Dog inherits from Mammal
public:
    void bark() {
        cout << "Barking...\n";
    }
};
int main() {
    Dog myDog;
    myDog.eat(); // Inherited from Animal
    myDog.breathe(); // Inherited from Mammal
    myDog.bark(); // Defined in Dog
}
```
### 4.Hierarchical Inheritance
- In this type of inheritance, multiple derived classes inherit from a single base class.
```cpp
class Animal {
public:
    void eat() {
        cout << "Eating...\n";
    }
};
class Dog : public Animal { // Dog inherits from Animal
public:
    void bark() {
        cout << "Barking...\n";
    }
};
class Cat : public Animal { // Cat inherits from Animal
public:
    void meow() {
        cout << "Meowing...\n";
    }
};
int main() {
    Dog myDog;
    Cat myCat;
    myDog.eat(); // Inherited from Animal
    myDog.bark(); // Defined in Dog
    myCat.eat(); // Inherited from Animal
    myCat.meow(); // Defined in Cat
}
```
### 5.Hybrid Inheritance
- In this type of inheritance, a derived class inherits from multiple base classes, and those base classes may also have their own base classes.
```cpp
class Engine {
public:
    void startEngine() {
        cout << "Engine starting...\n";
    }
};
class Wheels {
public:
    void rotateWheels() {
        cout << "Wheels rotating...\n";
    }
};

class Vehicle {
public:
    void start() {
        cout << "Vehicle starting...\n";
    }
};
class Car : public Vehicle, public Engine, public Wheels { // Car inherits from Vehicle, Engine, and Wheels
public:
    void drive() {
        cout << "Car driving...\n";
    }
};
int main() {
    Car myCar;
    myCar.start(); // Inherited from Vehicle
    myCar.startEngine(); // Inherited from Engine
    myCar.rotateWheels(); // Inherited from Wheels
    myCar.drive(); // Defined in Car
}
```
## 3.Polymorphism 
- It is the ability of a function or an object to take on many forms. In C++, polymorphism is achieved through function overloading, operator overloading, and virtual functions. It allows for code reusability and flexibility in programming.
```cpp
class Shape {
public:
    virtual void draw() const {
        cout << "Drawing a shape...\n";
    }
};
class Circle : public Shape {
public:
    void draw() const override {
        cout << "Drawing a circle...\n";
    }
};
class Square : public Shape {
public:
    void draw() const override {
        cout << "Drawing a square...\n";
    }
};
int main() {
    Shape* shape1 = new Circle();
    Shape* shape2 = new Square();
    shape1->draw(); // Calls Circle's draw()
    shape2->draw(); // Calls Square's draw()
    delete shape1;
    delete shape2;
}
```
- All About Polymerphism
        1.Compile time polymerphism
        2.Run time polymerphism
### 1.Compile time polymerphism
- It is achieved through function overloading and operator overloading. The decision about which function to call is made at compile time based on the arguments passed to the function.
```cpp
class Calculator {
public:
    int add(int a, int b) { // function overloading
        return a + b;
    }
    double add(double a, double b) {
        return a + b;
    }
};
int main() {
    Calculator calc;
    cout << calc.add(3, 4) << endl; // Calls int version
    cout << calc.add(3.5, 4.5) << endl; // Calls double version
}
```
### 2.Run time polymerphism
- It is achieved through virtual functions. The decision about which function to call is made at runtime based on the type of the object pointed to by a base class pointer or reference.
```cpp
class Shape {
public:
    virtual void draw() const {
        cout << "Drawing a shape...\n";
    }
};
class Circle : public Shape {
public:
    void draw() const override {
        cout << "Drawing a circle...\n";
    }
};
class Square : public Shape {
public:
    void draw() const override {
        cout << "Drawing a square...\n";
    }
};
int main() {
    Shape* shape1 = new Circle();
    Shape* shape2 = new Square();
    shape1->draw(); // Calls Circle's draw() at runtime
    shape2->draw(); // Calls Square's draw() at runtime
    delete shape1;
    delete shape2;
}
```
KeyFacts
- Polymorphism allows for writing code that can work with objects of different types through a common interface (base class).
- Virtual functions enable dynamic dispatch, allowing the correct function to be called based on the actual object type at runtime.
- Compile-time polymorphism is resolved during compilation, while runtime polymorphism is resolved during execution.

## 4.Abstraction
- It is the process of hiding the implementation details of a class and exposing only the necessary features to the user. It allows for creating a simplified interface for complex systems and promotes modular programming.
```cpp
class Car {
private:
    string engineType; // hidden implementation detail
public:
    car(string type):enginetype(type){}
    void start() { // exposed interface
        cout << "Starting the " << engineType << " engine...\n";
    }
};
int main() {
    Car myCar("V8");
    myCar.start(); // User interacts with the simplified interface
}
```
- Abstraction can be achieved through abstract classes and interfaces in C++. An abstract class is a class that cannot be instantiated and may contain pure virtual functions, while an interface is a class that contains only pure virtual functions.
```cpp
class Shape {
public:
    virtual void draw() const = 0; // pure virtual function
    virtual ~Shape() = default; // virtual destructor
};
class Circle : public Shape {
public:
    void draw() const override {
        cout << "Drawing a circle...\n";
    }
};
class Square : public Shape {
public:
    void draw() const override {
        cout << "Drawing a square...\n";
    }
};
int main() {
    Shape* shape1 = new Circle();
    Shape* shape2 = new Square();
    shape1->draw(); // Calls Circle's draw()
    shape2->draw(); // Calls Square's draw()
    delete shape1;
    delete shape2;
}
```
KeyFacts
- Abstraction allows for focusing on what an object does rather than how it does it, promoting separation of concerns.
- Abstract classes and interfaces provide a way to define a common interface for a group of related classes while hiding the implementation details.

# Few other things 
- `this` pointer: It is an implicit pointer that points to the current object of the class. It is used to access the data members and member functions of the class from within the class.
```cpp
class Person {
public:
    string name;
    int age;
    void introduce() {
        cout << "Hi, I'm " << this->name << " and I'm " << this->age << " years old.\n";
    }
};
```
- `friend` functions: These are functions that are not members of a class but have access to the private and protected members of the class. They are declared using the `friend` keyword inside the class.
```cpp
class Box {
private:
    double width;
public:
    Box(doublw w):width(w){}
    friend void printWidth(Box& b); // friend function declaration
};

void printWidth(Box& b) { // friend function definition
    cout << "Width of box: " << b.width << endl; // can access private member
}
int main() {
    Box myBox(5.0);
    printWidth(myBox); // Calls friend function
}
```
- `Operator overloading`: It is the process of defining custom behavior for operators when they are used with objects of a class. This allows for more intuitive and natural syntax when working with objects.
```cpp
class Complex {
public:
    double real, imag;
    Complex(double r, double i) : real(r), imag(i) {}
    Complex operator+(const Complex& other) const { // operator overloading
        return Complex(real + other.real, imag + other.imag);
    }
};
int main() {
    Complex c1(1.0, 2.0);
    Complex c2(3.0, 4.0);
    Complex c3 = c1 + c2; // Uses overloaded operator+
    cout << "Result: " << c3.real << " + " << c3.imag << "i\n";
}
```
# Some special facts I Know overall
- C++ supports multiple inheritance, which can lead to the "diamond problem" when two base classes inherit from the same class. This can be resolved using virtual inheritance.
- C++11 introduced move semantics, which allows for efficient transfer of resources from temporary objects without the overhead of copying.
- C++ supports both compile-time and runtime polymorphism, allowing for flexible and efficient code design.
- C++ allows for operator overloading, which can make code more intuitive but should be used judiciously to avoid confusion.
- C++ has a rich set of features for memory management, including smart pointers and RAII (Resource Acquisition Is Initialization) to help prevent memory leaks and dangling pointers.
### About Special keywords used in OOP
- `virtual`: Used to declare a member function as virtual, enabling runtime polymorphism.
- `override`: Used in a derived class to indicate that a member function is intended to override a virtual function in the base class.
- `final`: Used to prevent further overriding of a virtual function or inheritance from a class.
- `explicit`: Used to prevent implicit conversions for constructors and conversion operators.
- `constexpr`: Used to declare that a function or constructor can be evaluated at compile time.
- `friend`: Used to declare a function or another class as a friend, allowing it access to private and protected members of the class.
- `this`: An implicit pointer available in non-static member functions that points to the object for which the member function is called.
